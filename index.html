Whop embedded checkout (playground)

Run npm start and open this page from the dev server (not file://). The API key is sent only to your local server, which calls Whop — do not deploy this pattern to a public host without proper auth.

Environment

Match the embed: sandbox keys and prod_ / biz_ IDs must come from sandbox.whop.com when using sandbox.

Credentials & product
Customer

When set, the checkout email field is hidden and this value is applied (Whop does not allow changing the built-in “Join” label; we add your own pay button below).

Used for the visible pay button and sent as the plan title to Whop.

Billing (sent into checkout)

Values are prefilled on the embed. If all of name, line 1, city, state, postal, and country are filled, the billing block in the embed is hidden and the same data is pushed via Whop’s API after load.

Return URL

Local http://localhost is not accepted. Use a deployed page, or an https tunnel (e.g. ngrok) that can open this same playground or your real thank-you page.

Payment methods

Card is always enabled. In Whop sandbox, only card is supported; other choices are ignored.

Pricing

Dynamic totals use Whop’s checkout configuration with an inline plan (initial_price / renewal_price).

Webhooks

Register webhook destinations in the Whop dashboard (Developer → Webhooks). This playground does not register them via API.
